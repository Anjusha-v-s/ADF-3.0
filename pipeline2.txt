Pipeline Name

Scheduled

Purpose

The Scheduled pipeline acts as an orchestration and monitoring layer for the data ingestion process.
It triggers the Ingestion pipeline, passes runtime parameters, waits for completion, and sends execution alerts upon success.

This pipeline is typically used in scheduled or automated runs where monitoring and notifications are required.

High-Level Flow

Accept runtime parameters (schema, table, backdate)

Execute the Ingestion pipeline

Wait for ingestion completion

Trigger an alert via HTTP Web Activity

Pipeline Parameters
Parameter	Type	Description
schema	string	Source database schema name
table	string	Source table name
backdate	string	Override CDC timestamp for backfill
